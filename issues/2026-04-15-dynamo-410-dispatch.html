Bug Fixes

Twenty-Six Bugs, Most of Them About Pins

DynamoDS/Dynamo → Bug Fixes

Six separate connector pin bugs are resolved in one release: save-reload position drift, Save As data loss, copy-paste omission, drag-operation drops, undo failures, and orphaned note pins. Beyond pins: AppData versioning, autocomplete crashes, pan/zoom stability, and geometry GC.

  • Pin position drift on save/reload — fixed
  • Pins lost on Save As — fixed
  • Pins not included in copy-paste — fixed
  • Pinned notes orphaned on node delete — fixed
  • Pin undo/redo now fully reliable
  • AppData wrote to 10.0 folder instead of 4.1 — fixed
  • Two autocomplete crash paths closed
  • Two pan/zoom crash paths on large graphs fixed
  • Geometry GC: CLR-container objects no longer prematurely freed
  • Info + Warning bubbles now coexist on non-chart nodes
26
Bug Fixes
Stability · Pins · AppData
What This Means for You
PR #17018 is critical if you've been running 4.1 RC builds — user settings were written to a 10.0 directory and may not migrate automatically. Check %AppData%\Dynamo\Dynamo Core\ before upgrading production environments. The geometry GC fix (PR #16821) resolves a subtle crash class when downstream node output types change.

  • PolyCurve.ByGroupedCurves + Arc.ByStartEndAndTangencies in creation group

  • Scale-adaptive tolerance for surface stitching operations

Performance

One Hash Fix That Changed Everything

DynamoDS/Dynamo → Performance Improvements

Two targeted fixes with outsized impact. A severe geometry marshaling slowdown traced to hash collisions is eliminated by switching to identity-based hash codes. UI thread responsiveness improves through dispatcher deferral and view model caching.

  • Geometry marshaling: identity hash codes eliminate hash collision cascade (PR #16754)
  • UI: expensive operations deferred to the WPF Dispatcher
  • View models: null-safety and caching reduce redundant recalculation
2
Performance Fixes
Marshaling · WPF · Dispatcher
What This Means for You
PR #16754 is the sleeper hit of this release. If your graphs slow to a crawl with hundreds of geometry objects, the hash collision fix justifies this upgrade on its own. The dispatcher deferral pattern used in the view model fix is also a solid technique to borrow for your own WPF performance work in the broader Dynamo codebase.

Dynamo Dispatch · Editorial Verdict
A Release That Cleans Its Own House
4.1.0 is a dense, practical release. The pin bug sweep finally closes a long-running reliability gap that affected every serious graph author. The geometry marshaling fix is a sleeper performance win that could meaningfully speed up geometry-heavy workflows. The AI agent skills section signals a genuine investment in developer tooling — fourteen PRs to build a canonical, CI-validated skill directory is not a side project. Templates are now first-class citizens on the start page. PythonNet3 is unambiguously the baseline. This is not a glamorous release. It is a foundation release — one that makes the next big thing possible.
