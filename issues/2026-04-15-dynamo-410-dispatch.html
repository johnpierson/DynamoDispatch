Features

The Controls Are Finally Where You Left Them

Features

Dynamo's workspace and sidebar have historically been places where menus and panels appeared when called and disappeared when dismissed, with no memory of where you were. Features — Continued

The API Catches Up to How Libraries Actually Work

Features

When third-party developers publish Dynamo nodes, they often need to declare what category their nodes belong to. But the attribute controlling this — NodeCategoryAttribute — lived in DynamoCore, forcing an unnecessary tight dependency on a heavy package. This release type-forwards it to DynamoServices. ActionGroupDisposable is also now nestable, so complex multi-step operations can compose their undo groups without conflicts. And the DYN file format gets its first machine-readable specification.

1

API · DynamoServices · WebView2
#16884 — Inline Watch node deletion now rewires connected ports
What This Means for You
The NodeCategoryAttribute type-forward (PR #16914) is the most directly actionable change here. If you maintain a custom node package that imports DynamoCore only for category declaration, you can drop that reference after upgrading. The JSON Schema DYN spec is also worth bookmarking — it's the first time the file format has had a machine-readable contract.

Templates

The Start Page Finally Shows Its Work

Templates

Template cards in the start page previously showed just a name — no visual hint of what the template does. From 4.1.0, each card shows a thumbnail. The File → Open → Template dialog has also been tightened: it no longer accepts DYF (custom node) files, which could be opened through it accidentally. Placeholder icons cover items that don't have custom thumbnails yet, so the grid never appears broken.

Bug Fixes — Part I

Six Pin Bugs Walk Into a Release. None Walk Out.

Bug Fixes

Connector pins are the anchors you attach to wires to lock them in place. They're stored alongside the graph. Six separate ways this could go wrong were fixed in one release: position drift on reload, disappearing on Save As, missing from clipboard, undo not working, notes losing their pin, and a new transient-pin reconnection path that makes the whole system more robust under rapid edits.

6
Connector Pin Bugs Fixed in One Release
Pins · Clipboard · Undo
What This Means for You
The pin persistence fixes (PR #17018 area) are the most impactful here — if you've been working around pin-loss on Save As or save/reload drift, those workarounds can be dropped. The race condition crash fix during simultaneous move+delete is subtle but surfaces on fast input, especially in scripted or automation-driven graph editing workflows.
03
Bug Fixes — Part II

The Camera Drifts. The Gizmo Fails. Both Fixed.

Bug Fixes

Two related rendering failures: the viewport camera slowly drifted when the user was zoomed in far, and the transform gizmo stopped redrawing correctly. Both traced to a shared WPF BitmapCache being reused across threads — a subtle race that corrupted the render state. Pan/zoom crashes on large graphs came from the same root. This second batch also closes geometry GC issues where CLR containers holding geometry pointers could be freed before consumers finished with them.

26
Total Bugs Closed in 4.1
WPF · BitmapCache · Geometry GC
What This Means for You
The geometry GC fix (PR #16821) matters if you have nodes that hold LibG geometry references across async or multi-step execution — the previous behavior could silently return null or crash downstream. The AppData folder fix (PR #17018 area) is critical for anyone who ran 4.1 RC builds: settings may have landed in a 10.0 directory and won't migrate automatically.

Geometry & LibG

Curves That Finally Do What They Promise

Geometry & LibG Updates

Voronoi diagrams partition space by "which seed point is closest?" — Dynamo's implementation was failing on inputs where edge lengths varied widely. A relative edge-length filter now handles those cases. PolyCurve.ByPoints had undefined behavior at edge cases where it would return different types in different scenarios; the behavior is now unified and predictable. Periodic NURBS curves — closed smooth curves like circles — get new API exposure for their knot and control point topology.

0
Undefined PolyCurve Behaviors Remaining
LibG · NURBS · Tessellation
#16931 — New node icons for NurbsCurve.PeriodicKnots and PeriodicControlPoints
What This Means for You
The new periodic NURBS APIs (PR #16896) expose curve topology that previously required DesignScript workarounds. If your C# nodes wrap LibG geometry, the unified PolyCurve.ByPoints return type and concrete Curve.PullOnto* types may change downstream type checks — worth a regression pass on affected nodes before shipping to users.

Performance

One Low-Level Fix. Massive Speed Recovery.

Performance Improvements

Dynamo keeps a lookup table to find geometry objects fast. Every object needs a unique ID so the table knows which is which. The old method borrowed IDs from .NET's default system — which can hand the same ID to two different objects. When that happens in a large table, every lookup turns into a slow search instead of an instant jump. The fix is one line: use memory addresses as IDs instead. They're guaranteed unique. The second fix stops expensive UI calculations from blocking the drawing thread.

0
Geometry Hash Collisions After Fix
CLR · Dispatcher · LibG
#16754 — Identity hash codes eliminate geometry marshaling collision cascade
What This Means for You
PR #16754 is the sleeper hit of this release. If your graphs slow to a crawl with hundreds of geometry objects in play, the hash collision fix justifies this upgrade on its own. The Dispatcher deferral pattern used in the view model fix is also a solid technique to borrow for any WPF performance work you're doing in the broader Dynamo codebase.

PythonNet3

Purple Is Gone. PythonNet3 Is the Baseline.

PythonNet3

When Dynamo migrated its Python nodes to PythonNet3, the team added a distinctive purple syntax highlight as a visual signal: "this is the new runtime." That marker has served its purpose. 4.1.0 removes the purple highlighting — Python is now PythonNet3 by assumption, not by flag. The sample scripts bundled with Dynamo have been updated to match, replacing PythonNet2-style imports throughout.

0
Purple Migration Flags Remaining
PythonNet3 · Python · IronPython
#16615 — Purple syntax highlighting removed; PythonNet3 is now default
What This Means for You
If you maintain Python node documentation or sample scripts for Dynamo users, now is the time to audit for remaining PythonNet2-style imports (PR #16600). The purple highlighting removal (PR #16615) is cosmetic but marks PythonNet3 as the unambiguous baseline — update any screenshots or tutorials that show the old highlight color.
08
Build & Infrastructure

The Pipeline Catches Up to .NET 10

Build & Infrastructure

Post-RC, master was bumped to 4.2.0 and the performance test suite was upgraded to .NET 10. A patch to global.json changes the SDK roll-forward policy: instead of requiring an exact SDK version match, builds now accept any newer patch. This means .NET publishes a patch update without breaking local builds or CI. The npm registry reachability check was rewritten to use a reliable WebClient call after the previous Invoke-WebRequest approach proved flaky in CI environments.

0
Builds Broken by SDK Patch Updates
.NET 10 · CI · NuGet
What This Means for You
The rollForward change (PR #16815) in global.json means local builds now accept .NET 10 patch updates without a manual SDK pin. If you maintain CI scripts for Dynamo forks or downstream repos, the npm registry fix (PR #16834) is worth reviewing — the Invoke-WebRequest pattern it replaces may exist in your own pipeline scripts.

AI Agent Skills & Tooling

Skills Get a Canonical Home and a Validation Gate

AI Agent Skills & Tooling

Agent skills in the repository were scattered across tool-specific directories depending on which IDE or AI assistant had added them. This release canonicalizes everything under .agents/skills and adds a validate-skills CI check that fails the build if any tool's directory diverges from that path. New skills added: dynamo-unit-testing for structured C# test generation, a C# Janitor for automated code health checks, UX Designer, and Content Designer — alongside three experimental architecture agents.

7
AI Workflows Ready to Use Out of the Box
AI · CI · Claude · Copilot
What This Means for You
The validate-skills CI check (PR #16961) enforces consistency across IDE wrappers — if you contribute using Claude Code, Cursor, or VS Copilot, the canonical .agents/skills directory is your source of truth. The dynamo-unit-testing and C# Janitor agents are immediately applicable to day-to-day C# work on the Dynamo repo.
