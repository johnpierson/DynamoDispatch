05
Performance

One Low-Level Fix. Massive Speed Recovery.

DynamoDS/Dynamo → Performance Improvements

The geometry marshaling layer was using default CLR object hash codes, causing collisions when tracking large numbers of geometry objects simultaneously. The result: severe, non-obvious performance degradation that looked like a slow graph, not a bug. Identity-based hash codes eliminate it entirely.

  • Geometry marshaling: identity-based hash codes eliminate collisions
  • Dispatcher: expensive view model ops deferred with null-safety + caching
0
Hash Collisions After Fix
CLR · Dispatcher · LibG
What this means for you

If you have graphs processing large geometry sets — structural analysis, complex facade, MEP routing — benchmark them before and after upgrading. The hash collision fix may reveal previously masked slowdowns. Document timing differences; this fix could meaningfully change perceived performance in production graphs.

Editorial Verdict · Release 4.1.0
A Release Worth Every PR

4.1.0 is a solid, substantive release. The templates system is a genuine user-facing win; the connector pin fixes are overdue but thorough; and the geometry performance fix will quietly rescue workflows that users may have written off as just slow. The AI agent skills investment is the most forward-looking element — Dynamo is building structured AI tooling into the repo itself, which is unusual for an open-source CAD platform. No single change here is revolutionary, but the cumulative polish is real.
