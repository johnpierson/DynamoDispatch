01 — Features

Twenty-Two Features
That Quietly Improve
Everything

The headline feature count belies the cumulative impact. Node group styles are now customizable from Preferences — edit, remove, or reset them. Home and End keys navigate to the leftmost and rightmost nodes on the canvas. A new online status indicator (globe icon with LED) appears in the status bar. And the Documentation Browser gains an AutoSync mode that follows your selected node as you work.

Deleting an inline Watch node now rewires connectors rather than orphaning them — a fix so obvious in retrospect it's hard to believe it waited this long. The Extensions panel auto-expands when you activate it while minimized. And the Extensions menu finally appears alongside the other top-level menus where it always belonged.

  • Node group styles: edit, remove, reset from Preferences
  • Home / End canvas navigation shortcuts
  • Documentation Browser AutoSync setting
  • Online status indicator in the status bar
  • Watch node deletion rewires connectors
  • Dynamo graph file spec published as JSON Schema + Markdown
  • NodeCategoryAttribute type-forwarded to DynamoServices
  • ActionGroupDisposable made nestable for composable undo/redo

04 — Bug Fixes

The Pin Bugs
Are Finally,
Completely Gone

Six connector-pin bugs in one release. Notes orphaned when their parent node was deleted. Pins shifting upward after save/reload. Connector pins lost on Save As. Pins dropped during copy-paste. Pin-unpin undo broken with unpredictable behavior on repeat. Selection state not clearing after unpin. All fixed. Drag operations now use transient pins to preserve connectivity throughout.

Beyond pins: the AppData versioning bug that wrote to a 10.0 directory instead of 4.1 is patched. Two node autocomplete crashes fixed. Info and Warning bubbles can now coexist on non-chart nodes. Pan/zoom crashes on the WPF rendering thread resolved.

  • AppData version directory: 10.0 → 4.1 (Sandbox + C3D hosts)
  • Node autocomplete crash on Alt+F4
  • Node autocomplete crash: simultaneous move + delete
  • Pinned note orphaned on node delete
  • Pins shift upward after save/reload
  • Connector pins lost on Save As
  • Pins omitted from copy-paste
  • Pin/unpin undo broken (Ctrl+Z)
  • Pan/zoom: WPF rendering thread BitmapCache crash
  • Gizmo redraw failure + camera drift when zoomed
  • Geometry objects prematurely GC'd in CLR containers
  • AI Assistant invalid input values (Revit categories) handled
§ V — Geometry & LibG

Periodic NURBS Arrive.
Tessellation Improves.
Curves Unify.

LibG gains two new first-class APIs: NurbsCurve.PeriodicControlPoints and NurbsCurve.PeriodicKnots, with icons and documentation. Periodic NURBS curves are essential for closed, smooth surface modeling — having them as nodes removes a class of workarounds that computational designers have been patching around for years.
  • Periodic NURBS: PeriodicControlPoints and PeriodicKnots nodes, with icons + docs
  • Voronoi and Delaunay tessellation: relative edge-length filter + UV clamping for out-of-range input
  • PolyCurve.ByPoints and Polygon.ByPoints behavior unified
  • Curve.PullOntoPlane / PullOntoSurface now return concrete curve types (⚠ potential graph impact)
  • Scale-adaptive tolerance for surface stitching
  • LibG NuGet updated to 4.0.0.4313

06 — Performance
0
Hash collisions after the fix

One Low-Level Fix.
Massive Speed Recovery.

The geometry marshaling layer was using default CLR object hash codes, causing hash collisions when large numbers of geometry objects were tracked simultaneously. The result: severe, non-obvious performance degradation on high-node-count graphs — not a crash, just a graph that silently gets slower as it grows. The fix switches to identity-based hash codes and eliminates the collision entirely.

The second fix defers expensive view model operations to the Dispatcher with null-safety and caching, improving general UI responsiveness — particularly noticeable on large, complex workspaces.

AI Agent Skills & Tooling

The Repo Now Ships Its Own AI Workforce

Dynamo 4.1.0 includes a canonical .agents/skills directory with agent skills for unit testing, UX design (aligned to Autodesk's Weave Design System), content design (with hallucination guardrails and PR-based node descriptions), and C# code cleanup. A CI workflow — validate-skills — checks for drift between the canonical skills and IDE-specific wrappers. Wrappers sync to VSCode, Cursor, and Claude Code.

Three experimental architecture agents were also added: Codebase Patterns, Ecosystem Reviewer, and a modular skill writer. .github/copilot-instructions.md was expanded with comprehensive repository context and build commands.

It's not a user-facing feature, but it signals a significant shift in how the team expects contributors to work with the codebase. The infrastructure is versioned and shipped — not a side experiment.

09 — PythonNet3

Purple Is Out.
PythonNet3 Is In.

Python samples have been updated to use PythonNet3 APIs. Existing scripts using older PythonNet patterns may need review — check your custom Python nodes if you rely on PythonNet-specific interop patterns.

The purple syntax highlighting that appeared on Python nodes has been removed. A small change, but one that eliminates a persistent visual inconsistency that made Python nodes stand out from the rest of the canvas for the wrong reasons.

  • Python samples updated to PythonNet3
  • Purple syntax highlighting removed from Python nodes
